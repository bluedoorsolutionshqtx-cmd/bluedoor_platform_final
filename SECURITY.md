# Security Policy

## Supported Branches
Security fixes land on:
- prod (production)
- stage (pre-production)
- dev (development)

## Reporting a Vulnerability
Report security issues privately.
- Do not open public issues for vulnerabilities.
- Provide: steps to reproduce, impact, and any logs/screenshots.

## Handling
We will:
- confirm receipt
- assess severity
- patch and release
- publish a brief advisory when appropriate
